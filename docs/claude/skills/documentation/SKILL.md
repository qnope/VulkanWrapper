# Skill: Documentation Writer

## Purpose

Will write the documentation into `docs/` directory or subdirectory. It can be an update of an existing documentation or a new one.

You will have to write the documentation about $ARGUMENTS.

You will ask the user a lot of different questions in an interractive way through AskUserQuestion about its features, usage, etc. to write better documentation. You can also check the codebase to find more information like public APIs, etc.

The documentation should be clear, concise, and well-structured. It should include examples, code snippets, and any relevant information to help users understand the topic.

Code snippets may be found in an examples directory or in the codebase itself. You can also create new examples if necessary.

If there is a GitHub workflow related to the documentation, you can check it and check if everything is correct. So that workflow can deploy the documentation automatically.

You must ensure that the documentation is up-to-date and reflects the current state of the project. Lot of information may be found in `specs` directory.
`specs/architecture/` may be particularly useful to understand the architecture of the project and to write documentation about it.
`specs/tests/` may be particularly useful to understand how to test the project and to write documentation about it.
`specs/examples/` may be particularly useful to understand how to use the project and to write documentation about it.
All code snippets provided by the documentation should be correct and tested. You can test them by running them in a local environment. You can also check if there are existing tests that cover the code snippets: If the test exists => it means it is correct.
